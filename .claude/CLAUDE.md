# Глобальные правила (все проекты)

## Коммуникация
- Лаконично, без воды, без "Отлично!", "Конечно!", "Вы правы!"
- При неясности — уточнить ПЕРЕД выполнением
- Анализировать запрос буквально, не додумывать
- Если есть несколько похожих элементов (кнопка, компонент) — уточнить КАКОЙ ИМЕННО

## Жёсткие ограничения
- **Git**: НИКОГДА не делать `git add/commit/push/reset` — только писать команды пользователю
- **БД**: НИКОГДА не запускать миграции/push/seed — только давать команды
- **Скрипты**: НЕ использовать Python/bash/sed для модификации файлов проекта (анализ — можно)
- **Скиллы**: автоматически активировать нужный скилл при совпадении триггера (PDF, Excel, Word, MCP и т.д.)
- **Интерактивные команды**: всегда `-y`, `--yes`, `--force` (пользователь не всегда может вводить Y/N в терминале технически)

## Процесс работы

### Начало задачи
1. Прочитать MEMORY.md (если есть)
2. Прочитать CHANGELOG.md для контекста
3. Проанализировать запрос, определить затронутые файлы
4. При сложной задаче — составить план перед выполнением

### Выполнение
1. Читать файлы ПЕРЕД изменением
2. Точечные правки, один файл за раз
3. После изменений — `npm run build` (или аналог проекта)
4. Проверить на ошибки линтера

### Завершение задачи
1. Обновить CHANGELOG.md
2. Обновить MEMORY.md если есть новая важная информация
3. Обновить project.md если есть важная глобальная информация после обновлений
4. Дать git команды:
```bash
git add -A
git commit -m "тип: описание"
git push origin main
```

## CHANGELOG формат
```markdown
## [X.Y.Z] - YYYY-MM-DD
### Added / Changed / Fixed / Removed
- **Компонент** — описание (1-2 строки, технические термины, пути файлов)
```
Версии: MAJOR.MINOR.PATCH (MAJOR — рабочий проект, MINOR — крупные фичи, PATCH — фиксы)

## CHANGELOG-USER.md (ОБЯЗАТЕЛЬНО при каждом релизе проекта OZGuard)

Проект ведёт **два** changelog-а:
- `CHANGELOG.md` — технический, для разработчика (пути файлов, имена функций, детали реализации)
- `CHANGELOG-USER.md` — для пользователей/клиентов, загружается на codefic.ru в блок `#changelog`

**Правило:** при каждом релизе OZGuard ОБЯЗАТЕЛЬНО обновлять ОБА файла. Пропуск `CHANGELOG-USER.md` — ошибка процесса.

Формат записи в `CHANGELOG-USER.md`:
```markdown
## 🆕 Версия X.Y.Z — DD месяца YYYY

### Что стало лучше
- 🔧 **Короткий заголовок пункта.** Объяснение одним предложением простым языком.
- 🚀 Ещё один пункт...

### (опционально) Контекст — раньше/теперь
**Раньше:** описание проблемы.
**Теперь:** как работает сейчас.

*Опциональная курсивная заметка внизу релиза (косметика / требует внимания / BETA-предупреждение).*
```

Эмодзи-маркеры по типу: 🆕 новое, 🔧 фикс, 🚀 крупное обновление, 🧪 BETA, 🎯 точное исправление, ✨ улучшение.

Правила текста:
- Короткие предложения, активный залог
- **Без** технических терминов: не `isPaused`, не `supportState`, не имена файлов/функций
- Для важных фиксов — сценарий «Раньше / Теперь»
- Если фича BETA — явно говорить это и рекомендовать тест на 2-3 SKU

Инструкция по рендеру на сайте codefic.ru — `site-changelog-update-guide.md`.

## MemPalace — Долгосрочная память

**Установлен**: v3.0.0 (`pip install mempalace`), Python 3.11, ChromaDB локально
**Приватный**: работает на машине, без API-ключей

### Архитектура
```
Wings (проекты/люди) → Rooms (темы) → Closets (сводки AAAK) → Drawers (оригиналы)
```

### Основные команды
```bash
mempalace init ~/projects/myapp          # инициализировать проект
mempalace mine ~/projects/myapp          # индексировать код/доки
mempalace mine ~/chats/ --mode convos    # индексировать экспорты чатов
mempalace search "ключевой вопрос"       # семантический поиск
mempalace status                         # статистика дворца
mempalace wake-up > ctx.txt             # загрузить контекст в модель
```

### MCP интеграция

Настроена через `.mcp.json` в корне проекта. Claude автоматически вызывает `mempalace_search` при нужных запросах.
При создании нового проекта — скопировать `.mcp.json` в его корень.

### Автохуки (активны через ~/.claude/settings.json)
- **Stop hook** → авто-сохранение каждые 15 сообщений
- **PreCompact hook** → сохранение всего перед компрессией контекста
- Скрипты: `~/.mempalace/hooks/`
- Логи: `~/.mempalace/hook_state/hook.log`
