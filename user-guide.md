# OZGuard — Полная инструкция пользователя

> Этот файл предназначен для создания на сайте codefic.ru интерактивного блока с пошаговой инструкцией. Содержит все экраны, кнопки, сценарии и технические детали.

---

## Общее описание

OZGuard — расширение для Google Chrome, которое помогает продавцам OZON:
1. **Находить конкурентов** — по артикулу (SKU) товара показывает всех продавцов на той же карточке
2. **Подавать жалобы** — автоматически отправляет жалобы на нарушителей через чат поддержки seller.ozon.ru
3. **Экспортировать данные** — результаты в Excel, буфер обмена или напрямую в модуль жалоб

---

## Установка

### Шаг 1. Скачивание
- Скачайте архив расширения с сайта codefic.ru (кнопка «Скачать бесплатно»)
- Распакуйте ZIP-архив в любую папку

### Шаг 2. Установка в Chrome
1. Откройте `chrome://extensions/` в адресной строке
2. Включите **«Режим разработчика»** (переключатель в правом верхнем углу)
3. Нажмите **«Загрузить распакованное расширение»**
4. Выберите папку с распакованным расширением (где находится `manifest.json`)
5. Расширение появится в панели расширений Chrome (иконка щита)

### Шаг 3. Закрепление
- Нажмите иконку пазла в Chrome (справа от адресной строки)
- Найдите **OZGuard** и нажмите кнопку закрепления (pin)
- Иконка щита появится на панели для быстрого доступа

## Сценарий 1: Сканирование конкурентов

### Подготовка
1. Авторизуйтесь на **ozon.ru** в любой вкладке Chrome
2. Не закрывайте эту вкладку — расширение использует её для запросов

### Ручной ввод SKU

```
Таб: Сканирование
```

1. Кликните на **textarea** «SKU (по одному на строку)»
2. Введите артикулы товаров — каждый на новой строке:
   ```
   1234567890
   9876543210
   5555555555
   ```
3. Нажмите кнопку **«Найти конкурентов»**

### Пакетный импорт XLSX (бесплатно)

```
Таб: Сканирование → нижний блок «Загрузите XLSX»
```

1. Зайдите в **Личный кабинет OZON → Товары → Цены товаров**
2. Скачайте XLSX-файл с ценами
3. В расширении **перетащите файл** (drag-n-drop) на область «Загрузите XLSX шаблон» или кликните для выбора файла
4. Расширение автоматически извлечёт все SKU из файла
5. Нажмите **«Найти конкурентов»**

### Что происходит при сканировании

```
Визуальная схема процесса:

[Ввод SKU] → [Открытие окна OZON] → [Поиск товара] → [Переход на карточку]
     ↓                                                          ↓
[Прогресс-бар]  ←──────────────────────────────── [Парсинг продавцов]
     ↓
[Результаты + экспорт]
```

- Расширение **открывает отдельное окно** Chrome с OZON
- Для каждого SKU: поиск → переход на карточку → извлечение данных о продавцах
- Скорость: ~10 секунд на один SKU
- **Не закрывайте** рабочее окно OZON до завершения

### Элементы управления при сканировании

| Кнопка | Действие |
|--------|----------|
| **Найти конкурентов** | Запуск сканирования |
| **⏸ (Пауза)** | Приостановить (можно продолжить) |
| **⏹ (Стоп)** | Полная остановка сканирования |

### Прогресс

- **Прогресс-бар**: `3/15 SKU — 20%`
- **Лог**: сворачиваемый блок с таймстемпами — подробности каждого шага

### Результаты

После завершения появляется блок **«Результаты»**:

```
┌─────────────────────────────────────────┐
│ Результаты                    [12]      │
├─────────────────────────────────────────┤
│ SKU 1234567890 (Название товара)        │
│   ├── Продавец А — 1 500 ₽             │
│   ├── Продавец Б — 1 450 ₽             │
│   └── Продавец В — 1 600 ₽             │
│                                         │
│ SKU 9876543210 (Другой товар)           │
│   └── Продавец Г — 2 300 ₽             │
├─────────────────────────────────────────┤
│ [Скопировать] [Только SKU] [Excel]      │
│ [В жалобы] [Очистить]                   │
└─────────────────────────────────────────┘
```

### Кнопки экспорта

| Кнопка | Что делает |
|--------|-----------|
| **Скопировать** | Копирует результаты в буфер обмена (текстовый формат) |
| **Только SKU** | Копирует только уникальные SKU конкурентов (для жалоб) |
| **Скачать Excel** | Скачивает CSV-файл с разделителем `;` (открывается в Excel) |
| **В жалобы** | Передаёт SKU конкурентов в таб «Жалобы» для автоматической подачи |
| **Очистить** | Удаляет текущую сессию |

### Исключения при сборе

```
┌─────────────────────────────────────────┐
│ Исключения при сборе                    │
│ Продавцы из этого списка будут          │
│ исключены из результатов сканирования.  │
│                                         │
│ ┌─────────────────────────────────────┐ │
│ │ Мой магазин                         │ │
│ │ ООО Рога и Копыта                   │ │
│ │ BrandName                           │ │
│ └─────────────────────────────────────┘ │
│ [Сохранить]                             │
└─────────────────────────────────────────┘
```

- Вводите названия магазинов — **по одному на строку**
- Работает **частичное совпадение**: если написать `Рога`, исключит и «ООО Рога и Копыта», и «Рога»
- Нажмите **«Сохранить»** — настройки сохраняются и применяются при следующем сканировании
- Исключения действуют для всех будущих сканирований (сохраняются в хранилище расширения)

### Задержка между запросами

- Интервал (в миллисекундах) между обработкой каждого SKU
- **Рекомендации:**
  - 2000-3000 мс для 5-20 SKU
  - 4000-5000 мс для 50+ SKU (снижает риск блокировки)
- Нажмите **«Сохранить»** для применения

### Подсказки интерфейса

- Во вкладке «Жалобы» есть два баннера: VPN-предупреждение и инструкция «как работает бот».
- Каждый баннер можно скрыть кнопкой ✕. Состояние сохраняется.
- В Настройках есть две кнопки: **«Скрыть все подсказки»** — убрать разом, **«Показать все подсказки»** — вернуть если случайно скрыли.

### Жалобы: защита от каскада обращений

- **Автообращений за сессию** (по умолчанию **10**) — лимит новых обращений которые бот откроет без вашего подтверждения. После лимита ставит паузу с кнопкой «Продолжить». Можно поднять до 500 для больших каталогов.
- **Эскалаций подряд** (по умолчанию **5**) — если Ozon подряд эскалирует жалобы оператору, бот остановится и спросит разрешения продолжать. **0** = брендовый режим без остановок: бот работает с увеличенной паузой 20–35с между обращениями, подходит для каталогов на сотни брендовых тикетов.

### Жалобы: доказательства для отдельных SKU

```
┌─────────────────────────────────────────┐
│ Жалобы: доказательства для отдельных SKU│
│                                         │
│ [Ваш SKU (11-значный)]  [＋ Добавить]   │
│                                         │
│ ┌──────────────────────────────────────┐│
│ │ 1217169821   2 файл(ов)   [＋] [✕]  ││
│ │   certificate.pdf  809 KB            ││
│ │   unboxing.mp4     45 MB    [IDB]    ││
│ └──────────────────────────────────────┘│
└─────────────────────────────────────────┘
```

- Введите свой SKU → нажмите **＋ Добавить** → выберите файлы (image, video до 200 MB, PDF, DOC).
- Файлы <2 MB лежат в `chrome.storage.local`, крупные — в **IndexedDB** (бейдж `IDB`).
- Когда бот получает список жалоб с родительским SKU = ваш SKU, он прикрепит **эти** файлы, а не общий пул.
- Если SKU конкурента найден из нескольких ваших SKU — файлы объединяются (дедуп по имени+размеру).
- Если для SKU нет ни per-SKU файлов, ни общего пула — жалоба помечается **failed** и попадает в «Проблемные SKU».

**Быстрый способ добавления:** в таб «Сканирование» → «Результаты» → рядом с каждым вашим SKU есть кнопка **📎 К доказательствам**. Клик — переходит в Настройки с уже пред-заполненным вашим SKU, остаётся выбрать файлы. Если к SKU уже есть привязанные файлы, кнопка зелёная: **✓ Доказательства**.

## Сценарий 4: Автоматические жалобы (PRO)

```
Таб: Жалобы
```

### Подготовка
1. Авторизуйтесь на **seller.ozon.ru** (кабинет продавца) в любой вкладке
2. Подготовьте скриншоты-доказательства нарушений (необязательно, но рекомендуется)

### Пошаговая инструкция

#### Шаг 1. Ввод артикулов
- Введите SKU нарушителей в textarea (по одному на строку)
- Или нажмите **«В жалобы»** в табе Сканирование — SKU конкурентов передадутся автоматически

```
Предупреждения:
- 20+ SKU → "Большая очередь, процесс займёт время"
- 50+ SKU → "Рекомендуем разбить на несколько сессий"
```

#### Шаг 2. Выбор режима

| Режим | Описание |
|-------|----------|
| **Автоматический** | Бот сам навигирует, отправляет артикулы, файлы и ждёт результат. Полная автономия |
| **Тестовый (dry run)** | Только логирование — никакие действия не выполняются. Для проверки перед запуском |

#### Шаг 3. Выбор типа жалобы

Три варианта пути в дереве чата поддержки OZON:

| Тип | Путь в чате | Статус |
|-----|-------------|--------|
| **Плагиат моих карточек** | Личный кабинет → Кабинет бренда → Жалоба на товар/продавца → Плагиат карточек → Использование моих фото, видео, текста | Стабильный |
| **Использование моего контента (BETA)** | Товары и Цены → Контроль качества → Нарушение правил площадки другим продавцом → Использование моих фото, видео, текста | 🧪 BETA |
| **Использование моего бренда (BETA)** | Товары и Цены → Контроль качества → Нарушение правил площадки другим продавцом → Использование моего бренда | 🧪 BETA |

**BETA-режимы** добавлены в v5.9.15:
- Все действия логгируются с префиксом `[BETA]` для отслеживания
- Каждая навигационная кнопка имеет 3 retry-попытки перед пропуском шага
- После **5 SKU подряд в failed** бот автоматически ставит паузу с сообщением — возможно путь в Ozon изменился, пользователь решает «Продолжить» или «Остановить»
- При старте популярный баннер 🧪 предупреждает: «Режим BETA — протестируйте на 2-3 SKU»
- Рекомендуется: запускать BETA на небольших пакетах (5-10 SKU), пока не убедитесь что всё работает

#### Шаг 4. Прикрепление доказательств

Есть **два способа**, они комбинируются:

**A) Общий пул (для всех жалоб)**
- **Перетащите файлы** (drag-n-drop) или кликните по области «Доказательства — перетащите или нажмите»
- Поддерживаемые форматы: PNG, JPG, PDF, DOC, DOCX, MP4, MOV, WebM
- Размер до **200 MB**. Файлы >2 MB автоматически уезжают в IndexedDB (бейдж `IDB` рядом с именем)
- Бот прикрепит этот пул к жалобам для SKU, у которых нет персональных доказательств

**B) Per-SKU (для конкретного вашего SKU)**
- В Настройках → «Жалобы: доказательства для отдельных SKU» добавьте свой SKU и файлы для него
- Бот при обработке жалобы на конкурента узнаёт из какого вашего SKU найден нарушитель и прикрепит именно эти файлы
- **Приоритет выше** общего пула: если per-SKU файлы есть — они используются, общий пул игнорируется для этого SKU

**Если ни того ни другого нет** — SKU помечается `failed` с причиной «Нет файлов для прикрепления» и попадает в блок «Проблемные SKU».

#### Шаг 5. Запуск

1. Нажмите **«Начать»**
2. Бот автоматически:
   - Найдёт вкладку seller.ozon.ru
   - Перейдёт в чат поддержки (/app/messenger)
   - Пройдёт навигацию: Поддержка → Новое обращение → Личный кабинет → Кабинет бренда → Жалоба на товар
   - Для каждого SKU: введёт артикул → прикрепит файл → дождётся ответа → нажмёт «Пожаловаться на другой»

### Элементы управления

| Кнопка | Действие |
|--------|----------|
| **Начать** | Запуск автоматической подачи жалоб |
| **⏸ (Пауза)** | Приостановить бота |
| **⏹ (Стоп)** | Полная остановка |
| **Очистить** | Сброс текущей сессии жалоб |

### Статус-карточка

```
┌─────────────────────────────────────────┐
│ ⏳ Навигация по дереву чата...           │
│ Подсказка: не трогайте окно OZON        │
└─────────────────────────────────────────┘
```

Иконки статуса:
- ⏳ — В процессе
- ✅ — Завершено успешно
- ❌ — Ошибка (нужно ручное действие)

### Очередь SKU

```
┌─────────────────────────────────────────┐
│ ✓ 1234567890  (обработано)              │
│ ● 9876543210  (в процессе)              │
│ ○ 5555555555  (ожидание)                │
│ ✗ 7777777777  (ошибка — нет файлов)     │
│ 🛎 4444444444  (передан оператору)       │
└─────────────────────────────────────────┘
```

Статусы:
- ✓ **done** — жалоба принята, Ozon скрыл товар
- 🛎 **escalated** — чат передан оператору на ручное рассмотрение (обращение не закрываем, ждём ответа оператора)
- ✗ **failed** — техническая ошибка (нет файлов, зацикливание, поле ввода недоступно)
- ○ **pending** — в очереди
- ● **processing** — обрабатывается сейчас

### Блок «Проблемные SKU»

Появляется ниже очереди когда есть failed или escalated. Две категории:

```
┌─────────────────────────────────────────┐
│ ⚠ Проблемные SKU                    [▼] │
├─────────────────────────────────────────┤
│ 🛎 Передано оператору (3)               │
│   [📋 Копировать] [↻ В очередь]         │
│   [📥 XLSX] [✕ Очистить]                │
│ 1234567890  Передано оператору — ...    │
│ ...                                     │
│                                         │
│ ✗ Ошибки (2)                            │
│   [📋 Копировать] [↻ В очередь]         │
│   [📥 XLSX] [✕ Очистить]                │
│ 9876543210  Нет файлов для прикрепления │
└─────────────────────────────────────────┘
```

| Кнопка | Действие |
|--------|----------|
| 📋 Копировать | SKU в буфер обмена |
| ↻ В очередь | Перенести в поле ввода артикулов — для повтора (например, после добавления файлов) |
| 📥 XLSX | Скачать CSV с колонками SKU/Причина/Время |
| ✕ Очистить | Удалить все записи этой категории |

### Гейт лимита обращений

После **10 новых обращений** за сессию или **5 эскалаций подряд** (значения по умолчанию, настраиваются в Настройках) бот автоматически ставит паузу:

```
┌─────────────────────────────────────────┐
│ ⏸ Достигнут лимит обращений              │
│ Бот создал 10 новых обращений в чате    │
```

- **Продолжить** — разрешает ещё одну порцию (столько же сколько настроено). Счётчик эскалаций подряд сбрасывается.

## Технические детали (для разработчика сайта)

### Архитектура расширения

```
Chrome Extension (Manifest V3)
├── Service Worker (background)     — оркестратор: API, парсинг, лицензия, жалобы
├── Content Script: interceptor.js  — перехват запросов OZON (world: MAIN)
├── Content Script: support-auto.js — автоматизация чата (world: ISOLATED)
└── Popup UI (HTML/CSS/JS)          — интерфейс расширения
```

### Хранение данных
- Все данные хранятся локально в `chrome.storage.local`
- Лицензия привязана к устройству через UUID fingerprint
- Fingerprint сохраняется в `chrome.storage.sync` (переживает переустановку)
- История — последние 10 сессий

### Безопасность
- Расширение **не собирает персональные данные**
- Лицензия проверяется через защищённый API (HTTPS)
- Все операции выполняются в контексте вашего браузера
- Код расширения доступен для аудита (не обфусцирован)

### Антибот-защита
- Случайные задержки между действиями (±30% джиттер)
- Имитация человеческого поведения (паузы, скролл)
- Дополнительная пауза каждые 20 SKU при большом количестве
- Настраиваемый интервал между запросами

### Совместимость
- Google Chrome 88+ (или Chromium-based: Edge, Brave, Opera, Яндекс Браузер)
- Manifest V3 (актуальный стандарт)
- Windows, macOS, Linux

## FAQ

**Q: Расширение не находит конкурентов.**
A: Убедитесь, что вы авторизованы на ozon.ru. Откройте любую страницу ozon.ru и проверьте, что вы залогинены.

**Q: Бот жалоб остановился.**
A: Проверьте вкладку seller.ozon.ru — возможно, чат требует ручного действия. Перезапустите бота кнопкой «Начать».

**Q: Сколько устройств на один ключ?**
A: 2 устройства. Чтобы перенести на другое — нажмите «Деактивировать» на старом устройстве.

**Q: Ключ не активируется.**
A: Проверьте формат (OZG-XXXXX-XXXXX-XXXXX). Убедитесь, что есть интернет. Если лимит устройств — деактивируйте на другом.

**Q: Можно ли использовать в Яндекс Браузере?**
A: Да, расширение работает в любом Chromium-based браузере.
