Что такое OZGuard?

OZGuard находит всех конкурентов, которые продают те же товары что и вы на OZON. Вы вводите SKU своих товаров — расширение открывает каждый товар, собирает список всех продавцов и показывает их цены. После этого можно одним кликом отправить нарушителей на проверку через жалобы.

Сайт: codefic.ru

🔍 Сканирование
1
Введите SKU своих товаров в поле (по одному на строку). SKU — это числовой артикул товара на OZON, его видно в адресной строке карточки товара или в личном кабинете продавца.
2
Выберите режим сканирования: ⚡ Быстрый — прямые API-запросы без открытия окна (в 5-10 раз быстрее); 🐢 Медленный — открывает окно OZON и имитирует поведение пользователя (медленнее, но безопаснее для сессии). Нажмите «Найти конкурентов».
3
В Результатах вы увидите по каждому товару: название, количество конкурентов, их SKU и цены. Кнопка ⎘ рядом с вашим SKU копирует его. Ссылка «открыть на OZON» ведёт на карточку. Кнопка ⎘ рядом с «N конк.» копирует сразу все SKU конкурентов по этому товару.
4
Кнопка 📎 К доказательствам рядом с каждым вашим SKU переносит его в Настройки → блок «Доказательства для отдельных SKU». Если к SKU уже привязаны файлы — кнопка меняется на ✓ Доказательства зелёным. Так удобно за один проход по результатам собрать все доказательства.
⏸ Пауза и ⏹ Стоп работают прямо во время сбора — даже если popup закрыт, через плавающую панель на странице OZON.
📦 Пакетный импорт XLSX
1
В личном кабинете OZON перейдите: Товары → Цены товаров → Скачать шаблон.
2
Перетащите скачанный .xlsx файл в зону «Загрузите XLSX шаблон» или нажмите на неё.
3
Выберите нужные статусы товаров — «Продается», «Готов к продаже», «Не продается» — и нажмите «Применить». SKU попадут в поле ввода.
По умолчанию отмечен только «Продается» — чтобы не тратить время на снятые товары.

  • 📥 Excel — скачивает CSV-файл, который открывается в Excel: Мой SKU, Название, SKU конкурента, Продавец, Цена, Ссылка

⚙️ Настройки
  • Лицензия PRO — нужна только для автоматической подачи жалоб. Весь функционал сбора SKU и пакетного импорта XLSX бесплатен. Купить PRO: codefic.ru/#pricing
  • Исключения при сборе — названия своих магазинов, которые не попадут в результаты. По одному на строку, частичное совпадение. Например, напишите «Restoremind» — и все ваши магазины с этим словом будут скрыты.
  • Задержка — пауза между запросами к OZON (мс). Рекомендация: 2000–3000 при 5–20 SKU, 4000–5000 при 50+ SKU.
  • Подсказки интерфейса — кнопки «Скрыть все подсказки» / «Показать все подсказки» управляют баннерами во вкладке «Жалобы» (VPN-предупреждение, инструкция).
  • Жалобы: защита от каскада обращений — лимит автообращений за сессию (по умолчанию 10) и лимит эскалаций подряд (по умолчанию 5). После лимита бот ставит паузу с кнопкой «Продолжить». 0 эскалаций = брендовый режим без остановок, с увеличенной паузой между обращениями (для больших каталогов).
  • Жалобы: доказательства для отдельных SKU — добавьте свой SKU и файлы (видео до 200 MB, сертификаты, PDF). Бот при подаче жалобы на конкурента узнаёт из какого вашего SKU он был найден и прикрепит именно эти файлы. Если у вас несколько своих SKU указали на одного конкурента — файлы объединяются. Если для SKU нет привязанных файлов — используется общий пул из вкладки «Жалобы». Если и общего нет — SKU помечается failed.
📨 Жалобы PRO
1
Введите SKU конкурентов вручную, нажмите «В жалобы» из Сканирования, или загрузите Excel-файл кнопкой «Импорт из Excel».
2
Откройте seller.ozon.ru → Поддержка в браузере (вкладка должна быть открыта).
3
Выберите тип жалобы — три варианта:
  • Плагиат моих карточек — стабильный путь через «Личный кабинет → Кабинет бренда → Жалоба → Плагиат → Использование моих фото/видео/текста». Используется чаще всего.
  • Использование моего контента (BETA) — новый путь через «Товары и Цены → Контроль качества → Нарушение правил площадки → Использование моих фото, видео, текста».
  • Использование моего бренда (BETA) — тот же путь, но с финальной кнопкой «Использование моего бренда».
⚠ BETA-режимы: логи с префиксом [BETA], при 5 ошибках подряд бот автоматически остановится и попросит подтверждения. Рекомендуем сначала протестировать на 2-3 SKU.
4
Нажмите «📨 Начать». Бот сам откроет чат поддержки, пройдёт дерево меню выбранного типа, введёт каждый артикул и прикрепит файлы-доказательства (если загружены).
💡 Режим «Тестовый» — пробный запуск без реальных действий, чтобы убедиться что всё работает правильно.
📎 Доказательства — два режима:
Общий пул — файлы из зоны «Доказательства» во вкладке «Жалобы». Прикрепляются ко ВСЕМ жалобам.
Per-SKU — файлы из блока «Доказательства для отдельных SKU» в Настройках. Привязываются к конкретному вашему SKU и прикрепляются только к нарушителям, которые были найдены по нему. Приоритет выше общего пула.
Видео/сертификаты до 200 MB сохраняются в IndexedDB браузера, переживают закрытие popup и перезагрузку.
🛎 Блок «Проблемные SKU» — показывает две категории:
Передано оператору — обращение ушло на ручное рассмотрение (не ошибка, Ozon эскалировал);
Ошибки — failed по техническим причинам (нет файлов, поле ввода недоступно, зацикливание).
Для каждой категории: 📋 Копировать, ↻ В очередь (для повтора), 📥 XLSX, ✕ Очистить.
⏸ Гейт лимита — после 10 новых обращений за сессию или 5 эскалаций подряд бот ставит паузу. Жмите Продолжить, чтобы разрешить ещё порцию. Это защищает ваш seller-аккаунт от риска блокировки за массовое создание обращений.
❓ Частые вопросы
  • «SKU не совпал — OZON перенаправил» — OZON иногда показывает другой товар при поиске по SKU снятых с продажи товаров. Это нормально — расширение пропускает такие SKU.
  • Бот завис? — нажмите ⏹ Стоп. Если popup закрыт — используйте плавающую панель на странице OZON.
  • Не запускаются жалобы? — убедитесь что вкладка seller.ozon.ru открыта в браузере.
  • Ложный failed у файла? — если бот показал failed, но файл в чате реально есть и Ozon ответил «Скрыли товар» — это было в старых версиях. С v5.9.11 верификация смотрит и на ответ бота, и на факт прикрепления.
  • Нужна помощь? — codefic.ru
