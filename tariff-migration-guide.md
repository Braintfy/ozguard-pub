# Инструкция: миграция тарифа «Навсегда» → «Годовая подписка»

## Контекст

OZGuard — Chrome-расширение для продавцов OZON. Текущая система монетизации:
- **Месячная подписка** (`monthly`) — 2 990 руб/мес
- **Вечный доступ** (`lifetime`) — разовая покупка (УДАЛЯЕТСЯ)

Замена: тариф `lifetime` удаляется, вместо него вводится **годовая подписка** (`yearly`).

## Что нужно сделать

### 1. Серверная часть (codefic.ru API)

**Файл/эндпоинт**: `POST /api/license`

#### 1.1 Обновить типы лицензий
- Удалить поддержку типа `lifetime` для **новых** активаций
- Добавить тип `yearly` — лицензия на 365 дней с момента активации
- **Существующие** лицензии `lifetime` должны продолжать работать (backward compatibility)

#### 1.2 Обновить логику активации
При `action: "activate"`:
- Если сервер определяет тип ключа как `yearly` — устанавливать `expiresAt = now + 365 дней`
- Если сервер определяет тип ключа как `monthly` — без изменений (`expiresAt = now + 30 дней`)
- Если ключ старого типа `lifetime` — оставить как есть (вечный)

#### 1.3 Обновить логику валидации (`action: "validate"`)
- `yearly` — проверять `expiresAt`, считать `daysLeft`
- `monthly` — без изменений
- `lifetime` (легаси) — возвращать `daysLeft: null`, `type: "lifetime"`

#### 1.4 Ответ API — формат
```json
{
  "valid": true,
  "type": "yearly",          // "monthly" | "yearly" | "lifetime" (легаси)
  "expiresAt": "2027-04-04T00:00:00Z",
  "daysLeft": 365,
  "activatedAt": "2026-04-04T00:00:00Z"
}
```

- Обновить existing товар «Месячная подписка» если нужно

#### 2.2 Обновить webhook обработку
- При успешной оплате годовой подписки — генерировать ключ с типом `yearly`
- Обновить email-шаблоны (см. ниже)

### 4. Расширение OZGuard (Chrome Extension)

> Эти изменения уже подготовлены в расширении. Перечислены для справки.

#### 4.1 background/service-worker.js

### 5. Backward Compatibility

**КРИТИЧНО**: Пользователи с `lifetime` ключами НЕ должны потерять доступ.

- Сервер: `type: "lifetime"` возвращает `valid: true, daysLeft: null`
- Расширение: `lifetime` → «Вечная» (UI без изменений для этих пользователей)
- Новые ключи `lifetime` не генерируются, но старые работают бессрочно
